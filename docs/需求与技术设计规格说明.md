# 需求文档

## 1. 应用概述

- **应用名称**：GifMeme
- **应用描述**：面向海外用户的 GIF 动图与贴纸表情网站，界面采用 terminal-inspired, data-dense, Inter+monospace 设计风格，支持浏览热门动图、分类筛选、关键词搜索，用户通过 Google 登录后可收藏表情，所有表情内容通过 Klipy API 免费接口获取，后台提供极简统计功能。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 海外互联网用户，偏好使用 GIF 动图与贴纸表情进行社交表达

### 2.2 核心使用场景
- 用户浏览首页热门动图与热门分类，发现感兴趣的表情
- 用户通过关键词搜索特定表情
- 用户收藏喜欢的表情（触发 Google 登录），登录后查看收藏列表
- 管理员在后台查看全站及单产品的访问统计数据

---

## 3. 页面结构与功能说明

### 3.1 整体页面结构

```
GifMeme
├── 首页（Home）
│   ├── 顶部导航栏
│   ├── 搜索栏
│   ├── 热门分类标签区
│   └── 热门动图瀑布流展示区
├── 分类页（Category）
│   └── 指定分类下的表情列表
├── 搜索结果页（Search Results）
│   └── 关键词匹配的表情列表
├── 表情详情页（Detail）
│   └── 表情预览 + 收藏操作
├── 收藏页（Favorites）
│   └── 已登录用户的收藏列表
└── 后台统计页（Admin Dashboard）
    ├── 全站统计
    └── 单产品统计
```

### 3.2 顶部导航栏（全局）

- 左侧：网站 Logo + 名称「GifMeme」
- 中间：搜索框（支持关键词搜索表情）
- 右侧：
  - 未登录状态：显示「Sign in with Google」按钮
  - 已登录状态：显示用户头像 + 「Favorites」入口 + 退出登录选项

### 3.3 首页（Home）

**热门分类标签区**
- 展示预设热门分类标签，包括：cat、funny、cute 等
- 点击标签跳转至对应分类页
- 标签支持横向滚动，适配分类数量扩展

**热门动图展示区**
- 通过 Klipy API 免费接口获取热门表情数据
- 以瀑布流或网格形式展示
- 每个表情卡片展示：
  - 表情内容（GIF 动图或静态贴纸 Sticker）
  - 表情类型标识（GIF / Sticker）
  - 收藏按钮（点击触发登录或直接收藏）
- 支持分页或滚动加载更多

### 3.4 分类页（Category）

- URL 格式：`/category/:name`（如 `/category/cat`）
- 通过 Klipy API 按分类关键词获取对应表情列表
- 展示形式与首页热门动图区一致
- 页面顶部显示当前分类名称
- 支持分页或滚动加载更多

### 3.5 搜索结果页（Search Results）

- 用户在搜索框输入关键词后跳转至本页
- 通过 Klipy API 按关键词搜索表情
- 展示匹配结果列表，形式与首页一致
- 无结果时展示空状态提示：「No results found for 「keyword」」
- 支持分页或滚动加载更多

### 3.6 表情详情页（Detail）

- 点击任意表情卡片进入详情页
- 展示内容：
  - 表情大图预览（GIF 动图或静态贴纸）
  - 表情类型（GIF / Sticker）
  - 收藏 / 取消收藏按钮
- 收藏操作规则见第 4 节业务规则

### 3.7 收藏页（Favorites）

- 仅登录用户可访问，未登录访问时跳转至 Google 登录
- 展示当前用户已收藏的全部表情
- 支持取消收藏操作，取消后实时从列表移除
- 收藏列表为空时展示提示：「You haven't saved any favorites yet.」
- 表情按收藏时间倒序排列

### 3.8 后台统计页（Admin Dashboard）

- 访问路径：`/admin`，仅限管理员账号访问
- 管理员身份通过预设账号或环境变量配置，不对外开放注册

**全站统计模块**
- 统计指标：PV（页面浏览量）、UV（独立访客数）、Ref（来源渠道/Referrer）
- 支持时间维度：
  - 按日查看（选择具体日期）
  - 按自定义时间区间查看（选择起止日期）
- 数据以表格或简单图表形式展示

**单产品统计模块**
- 统计维度：针对单个表情（以表情 ID 为单位）
- 统计指标：PV、UV、Ref
- 支持时间维度：按日 / 按自定义时间区间
- 提供表情 ID 或关键词检索入口，定位目标表情后展示其统计数据

## 4. 业务规则与逻辑

### 4.1 表情类型定义
- **GIF**：动态表情，文件格式为 .gif
- **Sticker**：静态表情贴纸，文件格式为静态图片
- 所有表情数据均通过 Klipy API 免费接口获取，接口调用方式以 Klipy API 官方文档为准

### 4.2 收藏触发登录规则
- 用户未登录时点击任意表情的收藏按钮，弹出 Google 登录弹窗
- 登录成功后，自动完成本次收藏操作，无需用户重新点击
- 已登录用户点击收藏按钮，直接执行收藏或取消收藏

### 4.3 登录方式
- 仅支持 Google 授权登录（手动实现 OAuth 2.0，不使用第三方认证库）
- 必须包含 CSRF 保护：OAuth 授权请求必须携带随机 `state` 参数，验证回调时必须校验该参数匹配，防止跨站请求伪造攻击
- 登录状态通过 JWT Cookie（stateless）持久化，用户关闭页面后重新访问仍保持登录状态（直至主动退出）

### 4.4 统计数据采集规则
- PV：每次页面加载记录一次
- UV：基于用户唯一标识（登录用户取用户 ID，未登录用户取 Cookie ID）去重统计
- Ref：记录 HTTP Referrer 字段，标识流量来源
- 单产品 PV/UV/Ref：在表情详情页加载时触发记录，绑定对应表情 ID
- 统计数据存储与处理：使用 Cloudflare Analytics Engine

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| Klipy API 请求失败或超时 | 展示错误提示「Failed to load content, please try again.」，提供重试按钮 |
| Klipy API 返回空数据 | 展示空状态提示「No content available right now.」 |
| 用户未登录访问收藏页 | 跳转 Google 登录，登录后重定向回收藏页 |
| Google 登录失败或取消 | 保持当前页面，不执行收藏操作，不展示错误弹窗 |
| 非管理员访问后台统计页 | 返回 403 页面或重定向至首页 |
| 搜索关键词为空 | 禁止提交搜索，搜索框边框高亮提示 |
| 统计时间区间起止日期相同 | 视为按单日查询，正常处理 |
| 统计时间区间结束日期早于开始日期 | 前端校验拦截，提示「End date must be after start date.」 |

## 6. 验收标准

- 首页正确展示热门动图与热门分类标签，数据来源为 Klipy API
- 点击分类标签可跳转至对应分类页，并展示该分类下的表情列表
- 搜索功能可通过关键词返回匹配表情列表，无结果时展示空状态
- 未登录用户点击收藏触发 Google 登录弹窗，登录后自动完成收藏
- 已登录用户可在收藏页查看、取消收藏的表情
- GIF 与 Sticker 两种类型在卡片上有明确标识区分
- 后台统计页仅管理员可访问，非管理员访问被拦截
- 全站统计与单产品统计均支持按日及自定义时间区间查询
- PV、UV、Ref 三项指标数据正确采集与展示
- 界面整体采用 terminal-inspired, data-dense 设计风格，使用 Inter+monospace 字体，视觉风格统一

## 8. Must NOT Have 设计禁区

本项目设计与实现必须严格遵守以下禁区：

- **Must NOT Have** 像素艺术 / 像素风格设计元素 — 必须严格遵循 terminal-inspired, data-dense 设计语言
- **Must NOT Have** 第三方认证库如 NextAuth / Auth.js — 必须手动实现 Google OAuth 2.0
- **Must NOT Have** 服务端会话存储（KV 存储）— 必须使用 JWT Cookie (stateless) 认证
- **Must NOT Have** D1 数据库存储统计数据 — 必须使用 Cloudflare Analytics Engine
- **Must NOT Have** 浏览器指纹用于 UV 去重 — 必须使用 Cookie ID
- **Must NOT Have** 品牌不一致 — 必须统一使用 GifMeme 作为品牌名称，域名统一指向 gifmeme.org
- **Must NOT Have** OAuth 流缺少 CSRF 保护 — 必须包含 `state` 参数校验
