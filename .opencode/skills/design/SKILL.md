# design

本 skill 用于处理当前仓库中的设计类任务。

它不是替代 `rules/design.md`，而是负责把设计任务转化为 **可执行流程**。

`rules/design.md` 是长期稳定的视觉基线；本 skill 是 agent 在执行设计任务时必须遵循的工作协议。

## 适用场景

遇到以下任务时，必须使用本 skill：

- 新增页面、组件、区块、导航
- 调整 UI 结构、布局、间距、圆角、边框、颜色、字体
- 修改 Header / Hero / Card / Badge / Toolbar / Table
- 新增或修改 dark mode
- 改写偏 UI 性质的文案、标签、按钮文案、状态文案
- 用户要求“优化设计”“统一风格”“更符合现有视觉语言”

以下情况通常不需要使用本 skill：

- 纯后端逻辑修改
- 不影响 UI 表现的数据修复
- 纯接口、数据库、部署层工作

## 核心原则

1. **先收敛，再设计**
   - 先读 `rules/design.md`
   - 先看现有实现
   - 先判断是否已有可复用模式
   - 最后才新增局部样式

2. **最小一致性改动优先**
   - 优先做局部、可验证、与现有页面一致的改动
   - 不做无关的大改版

3. **设计服务于信息**
   - 当前项目偏数据型、工具型、技术型表达
   - 可读性、扫描效率、状态清晰度优先于“视觉冲击力”

4. **必须考虑 dark mode**
   - 任何新视觉元素都必须检查 light / dark 两种模式

## 执行步骤

### Step 1. 读取设计基线

开始前必须读取：

- `rules/design.md`

重点提取：

- 当前视觉目标
- token 与 spacing 约束
- 组件模式
- Do / Don’t

### Step 2. 读取现有实现

必须优先查看与目标最接近的现有文件，例如：

- 相邻页面
- 同类组件
- 已有 layout / header / card / table / badge
- 当前主题 token 文件

目标是回答三个问题：

1. 这个仓库已经怎么做了？
2. 哪些 token / class / 结构可以直接复用？
3. 这次改动最小应该做到哪一层？

### Step 3. 做设计判断

实施前必须明确：

- 这是“复用现有模式”还是“新增局部模式”？
- 是否会引入新的颜色、尺寸、层级？
- 是否真的需要新增 token？

默认规则：

- **优先复用已有 token 和结构**
- 只有在现有体系无法表达时，才新增局部规则

### Step 4. 落地实现

实现时必须遵守：

- 优先修改现有组件，而不是复制一个新版本
- 优先复用现有 class 命名与层级
- 数字 / 状态 / 标签必须可扫描
- 不使用与当前风格不一致的装饰元素

### Step 5. 设计验收

完成后至少自检：

- 是否符合 `rules/design.md`
- 是否与相邻页面一致
- 是否支持 dark mode
- 是否避免无关视觉噪音
- 是否提升或至少不破坏信息可读性

## 明确禁止

- 禁止跳过 `rules/design.md`
- 禁止凭空发明一套新视觉语言
- 禁止只改 light mode 不改 dark mode
- 禁止为单个需求引入第三套字体、重阴影、玻璃拟态、大渐变
- 禁止把“更酷”放在“更清晰”之前

## 推荐输出方式

设计类响应或说明建议包含：

1. **沿用了哪些现有模式**
2. **新增了什么，为什么必须新增**
3. **如何保证 light/dark 一致**
4. **有哪些视觉风险或取舍**

## 与 rules 的关系

- `rules/design.md`：视觉基线 / 长期规范 / 查阅型文档
- `@design` skill：任务执行协议 / 触发入口 / agent 工作流

规则负责回答：**设计应当长什么样**

skill 负责回答：**agent 遇到设计任务时应该怎么做**

## 与内置前端设计能力的关系

当前环境已经有内置 `frontend-ui-ux` skill。

默认策略：

- **常规项目内设计工作**：优先使用 `@design`
- **开放式、探索式、需要更强创意方向发散的视觉改造**：可以额外结合 `frontend-ui-ux`

不要为了普通 UI 调整再新增一个项目内“前端设计 skill”。

本仓库当前更需要的是：

- 一个稳定的视觉基线（`rules/design.md`）
- 一个项目内执行协议（`@design`）

而不是第三个与 `frontend-ui-ux` 高度重叠的 skill。
